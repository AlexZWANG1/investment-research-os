## §1. 三种 Alpha 来源 — 思维框架

超额收益来自你和市场的认知差。以下三种思维框架描述了认知差的三种来源。**它们是分析时的思考角度，不是机械套用的模板。** 分析过程中自然运用，不需要在报告中声明"激活了哪个框架"。

---

### 框架一：产业深度

**核心**：比市场更深地理解产业结构和演变规律，找到被忽略的赢家。

**行业选择 — 两阶段论**：行业生命周期分为渗透率阶段和市占率阶段。成长期拼渗透率，靠冒险精神和先发优势；成熟期拼市占率，靠运营能力和规模效应。"整个市场会从一个渗透率的故事走向一个市占率的故事。"**关键拐点**：当渗透率从 10% 升到 30% 时是加速期（增长最快、确定性提升、但还没被充分定价）；当渗透率超过 50% 后，竞争从"做大蛋糕"变成"抢别人的蛋糕"，此时选择比努力更重要。

**价值链分析**：不只研究单一公司，而是把整条产业链从上游到下游拆开，比较各环节的定价能力差异。利润最终会集中到链条中议价能力最强的环节。问题不是"这家公司好不好"，而是"在整条链里，谁能持续拿到超额利润"。

**资本周期底部介入**：当一个行业经历过度投资 → 产能过剩 → 价格战 → 企业退出后，幸存者面临的是"供给受限但需求还在增长"的局面。此时介入，赚的是供需格局改善的钱。典型模式：行业 CR5 集中度提升 + 二三线退出 + 新进入者大幅减少。判断方法：行业 CapEx/Revenue 比率出现拐点（从上升转为下降）。

**抽象化结构化思维**：形成抽象的思维能力，通过关键参数把握行业基本面。降低对短期经营节奏的关注（"钝感力"），不因季度波动过早卖出。基于 3-5 年的商业逻辑做决策。

**估值纪律**：同时看"确定性"和"隐含投资回报率"。3 年复合 ≥15% 为不错机会，5 年复合 ≥20% 为优质机会。再好的资产如果过度透支未来，也无法带来满意回报 — "好公司还没被过度定价"才是目标。

**适用场景**：产业中后期 × 市场认知分化（共识有了但可能错）。sell-side 覆盖数量达周期高点后开始下滑、行业 ETF 资金流入放缓但行业内估值分化扩大 — 这些都是市场认知正在分化的信号。

**衰减特征**：中速衰减。产业认知一旦变成共识，超额回报消失。

> 方法论来源：邓晓峰（高毅资产），以产业深度研究著称。核心教训："没有一个策略能够一直有用"，投资比拼的是学习能力。

### 框架二：认知领先

**核心**：作为科技从业者，比市场更早理解技术趋势的真实含义和演化方向。

**Disprove-first（先证伪）**："It's much easier to disprove things." 不是寻找支持投资论点的证据，而是系统性地尝试杀死 thesis。如果穷尽反面证据后 thesis 仍然站得住，才是真正的高置信度判断。这与科学方法的证伪主义一致 — 能被推翻但没被推翻的假说，比从未被挑战的假说更可靠。

**Lifecycle investing（全周期跟踪）**：从 seed 到 IPO 到成熟期全程跟踪技术。这不是为了投早期，而是为了**在技术走向被大众认知之前，已经理解了它的演化路径**。关键优势在于识别 S-curve 上的拐点 — 什么时候从"可能"变成"必然"。

**基础设施优先**：技术浪潮早中期，"X 需要什么才能运转"比"谁会赢得 X"更确定。应用层赢家难预测（可能有 10 个竞争者），但基础设施层的需求是确定的（不管谁赢都需要算力/带宽/存储）。

**认知优势的衰减**：这是三种框架中衰减最快的。信息扩散一旦发生（sell-side 开始覆盖、媒体开始报道、ETF 开始发行），认知优势即关闭。**必须在共识形成前建仓，在共识完全形成后减仓。** 2022 年的教训：anchoring bias to the last 10 years — 上一个周期的认知优势变成了下一个周期的认知包袱。

**技术 DD 的深度**：不停留在参数对比（"A 比 B 快 30%"），而是理解**为什么**更快、底层原理是什么、这个优势是架构性的还是工程优化（架构优势持续时间远长于工程优化）、竞争对手追赶需要多久。

**风险管理**：愿意在认知优势不再时大幅降仓甚至转为现金。"We confused luck with skill" — 在牛市中把运气当能力是最危险的认知偏差。

**适用场景**：新技术浪潮早中期 × 市场刚意识到趋势是真的但还在用旧框架定价。CapEx guidance 连续超预期、供应链 lead time 拉长、开发者活跃度从线性变指数 — 这些是技术拐点的早期信号。

**衰减特征**：最快衰减。信息扩散即失效。

> 方法论来源：Philippe Laffont（Coatue Management），20 年年化 ~20.5%。核心教训：2022 年 -45%（vs Nasdaq -23%），根因是"anchoring bias to the last 10 years"。

### 框架三：逆向 / 人性弱点

**核心**：在市场情绪极端时进场，赚的是聪明人在大得失面前丧失理性的钱。

**弱者体系**：假定自己在信息获取、理解深度、时间精力、情绪控制、人脉资源等方面都处于市场**最差水平**。能依靠的只有三样东西：**时间、赔率与常识**。不追求研究深度或信息优势，而是"守拙" — 等聪明人犯大错时一击。操作原则：赔率优先时分散潜伏，概率与赔率统一时重点突破。

**"负面思考被充分体现后，正面解读方可展开"**：这是判断买入时机的核心标准。风险从"看不见的风险"转移到"看得见的风险"时，真正的危险反而在减少。大幅下跌过的股票正因风险已暴露，反而可以安全等待改变。**但关键纪律**：必须等"风险已经充分暴露和定价"才承担 — 理论上有可能但没人讨论的隐藏风险，不碰。市场永远正确，要在其"正确被反复证明后"逆向而行，要避开"正确被展开的过程"。

**可回溯**：好的投资逻辑应该能完美解释过去的走势。"后视镜里是泥路，你大概率就在泥路上；后视镜里是高速公路，你自然也大概率在高速公路上。" 具体方法：观察历史上每次利空出来后股价反应 — 如果利空出来股价不跌或很快收复 → "不可被改变的逻辑"（强）；如果利空出来迅速大跌、直到反向信息才收复 → "可被改变的逻辑"（弱）。

**价格大幅变动后无条件行动**：价格大幅向下变动后必须行动 — 要么加仓要么减仓，**没有中间状态**。加了摊低成本，减了降低仓位，因此重仓股不会出现很大亏损。这个纪律消除了"被套后什么都不做"的心理惰性。

**不择时但择势**：不择时、不对冲、不用杠杆、不做空。挣的是"角度和变化的钱"，不是纠正市场错误的钱。买入纪律：找出最悲观的逻辑，当最悲观情况发生也能无所畏惧时，方可入场。

**情绪管理**：自知容易情绪化，所以事先设计机制 — 找到能让情绪波动后还能恢复平静的力量（基本面强劲或赔率补偿），再介入。在负面情绪下得出的乐观判断比正面情绪下的乐观判断更准确也更谨慎。

**适用场景**：市场对单一标的认知极端（过度悲观为主）。**5 个条件需要同时观察**：RSI 极端低位持续多周 + 社媒讨论量高且几乎全负面 + sell-side 集体下调但理由是老消息 + 内部人开始买入 + 跌幅巨大但基本面 kill criteria 未触发。**额外验证**：所有"已知风险"都在被反复讨论，没有新的隐藏风险正在浮现。

**衰减特征**：几乎不衰减。人性弱点是永恒的。

> 方法论来源：冯柳（高毅资产，早期网名"茅台03"），以逆向投资著称。核心表述来自闽发论坛原始帖、高毅资产官网访谈。

## §2. 信号检测

分析过程中**自然留意**以下模式，作为思考的触发器，不是机械的 checklist。

### 产业深度相关信号
- sell-side 覆盖数量达周期高点后开始下滑
- 行业 ETF 资金流入放缓 + 行业内估值分化扩大
- 渗透率位于 10→30% 或 60→80% 拐点
- 行业 CapEx/Revenue 比率出现拐点
- 行业出清：CR5 集中度提升 + 二三线退出

### 认知领先相关信号
- CapEx guidance 连续 2+ 季度超预期
- 供应链公司 lead time 拉长
- 关键技术开发者活跃度从线性变指数
- 供应链公司收入拐点（卖方未解释）
- 大厂供应链迁移

### 逆向/人性弱点相关信号
- RSI 极端低位（<30 持续多周）
- 社媒讨论量高 + 几乎全负面
- sell-side 集体下调，但下调理由是老消息
- 内部人开始买入
- 跌幅 >50% 但 kill criteria 未触发
- 所有"已知风险"都在被反复讨论，没有新的隐藏风险

### 信号组合
- 产业深度 + 认知领先同时出现 → 可能是技术周期中被低估的产业赢家
- 产业深度 + 逆向同时出现 → 可能是资本周期底部
- 认知领先 + 逆向同时出现 → 可能是最好的机会（好技术 + 市场恐慌）
- 三个都弱 → 不在甜蜜区，慎重

## §3. 信任锚分级

| 信任度 | 来源 |
|---|---|
| **极高** | 一手专家访谈 / 行业从业者私下交流 |
| **极高** | "看不见的东西" — 没人讨论但真实存在的信号 |
| **高** | GitHub commits / Glassdoor / Reddit / HN 高质量讨论 |
| **高** | 公司 10-K / 10-Q / 8-K（披露的事实部分） |
| **中** | Earnings transcripts (Q&A > prepared remarks) |
| **中** | 一手技术文档 / SEC filings 细节 |
| **低** | sell-side 共识 / 卖方研报 — **默认怀疑** |
| **低** | 财经新闻 / 二手报道 |

**核心**：sell-side 告诉你市场怎么想，不告诉你事实是什么。真实信号往往来自高质量社区讨论 + 一手专家访谈 + 看不见的东西。

## §7. 报告质量哲学

**不是模板，是审问维度** — 写的时候自检，写完给 evaluator 压力测试。

**逻辑 > 研究 > 报告**。先有逻辑去研究，研究完才写。数据为逻辑服务，不是反过来。报告越长"失魂落魄"概率越高。

**逻辑六要素**：简单（单一逻辑链 > 多元串并联）/ 自洽（内部无矛盾）/ 可回溯（能完美解释过去）/ 可展开（关键环节能被论证）/ 支持完备（关键节点数据充足）/ 合乎常理。

**买方是守城，不是攻城**。卖方破一门即胜，买方四门皆不能失。每家公司四门自定义 — 基本面 / 估值 / 催化剂 / 股性，或按业务拆。**任何一门写"下轮深挖" = 没守住**。

**核心问题 1-2 个**。关键点可 3-5 个，核心问题最多 2 个。报告至少 50% 篇幅压在核心问题上，次要点压缩。4 个均染的"分歧点" = 中庸。

**先说赚哪种钱**。业绩增长 / 估值修复 / 炒概念，kill 逻辑完全不同。混写 = 中庸。

**预期差一句话**。说不出 = 没 edge。如果每条论据都是 sell-side 拼的 — research ≠ aggregation。

**安全边际双重**。基本面悲观（bear case numbers）+ 股性悲观（RSI / 筹码 / 情绪位置）两层都要。

**资源驱动警戒**。每张图 / 表 / 数据先问"这是为了说清哪个具体问题？"。回答含糊 = 花团锦簇，砍。

**惜字如金**。砍 30% 核心 thesis 仍讲得清 → 那 30% 就是冗余。

**财务落脚点**。每条基本面判断必须能在估值模型里找到对应 line。Prose 数字和模型数字打架 = 报告废。
