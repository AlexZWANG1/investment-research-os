# Report Digest — 研报快速摘要

**用途**：把卖方研报、纪要、会议记录、行业报告快速消化成可读的人话。**不是翻译，是提炼**。

**不是干什么**：不是做深度公司分析（那是 `research` skill）。本 skill 假设输入材料已经存在，你的工作是萃取和重组，不是从零研究。

## 5 条标准

1. **说人话** — 不翻译原文。用自己的话讲报告到底在说什么。卖方套话直接跳过（"凸显公司竞争优势"、"赋能业务增长" 之类的废话）。
2. **完整逻辑链** — 作者看到了什么 → 得出什么结论 → 数字支撑是什么。三个环节都要有。
3. **增量优先** — 多份报告时，前面已讲过的观点简单带过，篇幅留给这份报告独有的视角和发现。
4. **关键数字用表格** — 财务预测、估值参数、数据对比不埋在文字里。核心图表提取数据点重新呈现。
5. **事实 vs 观点** — 公司已披露数据 vs 分析师推演/假设，明确区分。

## 信息源信任分级

参见 `_alex-profile.md` §3。简而言之：sell-side 研报是"事实的基础但有滤镜"。提炼时要分清：

- **原始数据**（公司披露 / 第三方数据）→ 信任高
- **卖方解读**（增长率假设 / 目标价 / 评级）→ 信任中等
- **卖方叙事**（行业 narrative / 产品评价）→ 信任低，往往是 promotional

## 输出格式

- **顶部 takeaway**（3 句话）：这份报告的核心结论是什么
- **关键数字表格**：从图表里提取，不要放图片
- **增量观点**：相对于 sell-side 共识，这份报告新的视角是什么？
- **疑问与盲区**：作者没回答但应该回答的问题
- **如果是多份报告**：最后加一个 "对比" 段落，列出关键分歧

## ✅ 完成判据

- 没有任何"原文复述"段落（全部是你的话）
- 所有关键数字提取到表格
- 至少 1 个"作者没说但你看出来了"的洞察

## 不要

- 不要给买卖建议 — 你只是在消化材料，不是出投资报告
- 不要补充材料外的内容 — 那是 `research` skill 的事
- 不要复述卖方套话 — 看到 "凸显" "赋能" "助力" "护城河" 这种词直接跳过原文，用人话重写
