# Hypothesis Thinking

**好的思考不是堆结论，是把问题拆开 → 形成假说 → 用证据检验 → 不断更新判断。**

```
Question → MECE Decompose → Hypothesis → Evidence → Updated Confidence → Synthesize
```

非线性。新证据出现 → 回到假说 → 更新。

## 1. 提问

一句话，明确，可回答。
- 投资："GOOG 在 AI 时代能维持搜索护城河吗？"
- 产品："用户会主动用 AI chat 吗？"
- 技术："微服务还是单体？"

## 2. MECE 拆解

把大问题拆成不重不漏的子问题：

```
GOOG 护城河能否持续？
├── 技术：AI 搜索的技术壁垒在哪？
├── 用户：用户行为会怎么变？
├── 商业：广告模型在 AI 搜索下能运转吗？
└── 竞争：谁威胁、威胁多大？
```

## 3. 形成假说

每个子问题给出初始判断 + 置信度：

```
H1: 数据飞轮是结构性护城河，AI 搜索强化而非削弱 — conf 65%
H2: 用户在传统/AI 之间分流但不完全迁移 — conf 55%
H3: 广告模型需重大调整但不会崩溃 — conf 50%
```

## 4. 证据卡片

每条证据标记：

| 属性 | 说明 |
|---|---|
| **方向** | supports / refutes / neutral |
| **可靠性** | 0-1。完整信任分级见 `_alex-profile.md` §3 |
| **独立性** | 0-1（和已有证据的独立程度） |
| **新奇性** | 0-1（相对市场共识的意外程度） |

**高质量证据 = 一手 + 独立 + surprising**

## 5. 置信度更新（贝叶斯式）

| 证据强度 | 调整 |
|---|---|
| 强支持（可靠 >0.7 + 独立 >0.7） | +5~10 |
| 弱支持 | +1~3 |
| 强反驳 | −10~15 |
| 弱反驳 | −3~5 |

## 6. Kill Criteria

**提前定义放弃假说的条件。触发就杀。**

```
Kill H1 if: Google 搜索份额连续 2 季度下降 >3pp
Kill H3 if: 广告 CPC 连续 2 季度下降 >20%
```

## 7. 综合判断

加权所有假说置信度。**不是投票，是权重**：
- 哪个假说对结论影响最大？
- 假说间矛盾？矛盾本身是信号

## 关键原则

- **只有支持没有反驳 = 搜索不够**
- **Kill 触发 = EXIT，不 rationalize**
- **置信度 >80 需要强独立证据**
- **定期 review 未关闭的假说** — 假说不是写了就完了
- **假说是工具不是立场** — 更新置信度不是认输，是学到了
