# _log 标准

_log.md 是 Alex Wang 的投研**入口 + 思考日记**。三合一：

1. **入口** — Alex 打开 wiki 的首页。扫一眼知道 book 长什么样。
2. **导航** — 告诉 Alex 今天哪些 signal 值得深读，哪些不用管。
3. **日记** — 替 Alex 看了材料 + AI 的真实思考。看完 log 他不用自己翻 source 也知道发生了什么。

注意顺序：入口/导航是 **user-facing**（Alex 视角），日记是 **author-facing**（AI 内省）。两者分层呈现，不混写。

## 核心原则

### 1. 顶部入口必须"替 Alex 看过全景"

- **每个 signal 一行**：conviction 粗体在最前 + [[slug|短名]] + 一句话 tagline + position 指向
- **tagline 要具体**：不要"模型 commoditize"，要"Model + Harness 双层 commoditize → long 入口+inference infra，avoid 纯 model 估值"
- **分层分组**：按 thesis 主题（AI 基建 / 宏观 / 估值行为），不按 conviction 排序 — 让 Alex 看到相关性
- **相关性提醒**：在每组开头写 `— core: {共享假设}`，提醒 Alex 这组 signal 如果假设错一起受损

### 2. "今天重点" 是 Alex 最常用的段

Alex 平时看 _log 就是想知道 **"今天哪 2-3 个值得我点进去读"**。这段不是动作摘要表，是**推荐阅读清单**：

- 挑本轮 conviction 变化最大 / 改动性质最本质 / 最可能改变 Alex 下一步行动的 2-4 个
- 每个写 1-2 句说清楚**改动的本质**（不是数字变了多少，是叙事/机制变了）
- ✅ "gold 讲故事角度变了 — 之前混讲央行 structural 和通胀对冲，现在必须分两层"
- ❌ "gold 70→72，新增 2 篇 source 验证"

### 3. 日记每段一个发现 — 四层分层

**读了什么 → 他们说了什么 → 为什么重要 → 我的想法**

四层用粗体小标题分开，每层短（1-3 句）。**不要把多个 source 的发现写在一段里**，拆开做多段。

**关键：facts 不复述**。source/signal 页已写过的具体数字、引用、量化，log 里用一句话带过即可。真正展开的是"**我的想法**"层 — 内省、不确定、防御反应、bias 承认。

### 4. 我的想法必须真实

- ✅ "我读到的第一反应是防御性的 — 想找理由说这个数据有缺陷"
- ✅ "他自己创业说 Meta 碾压对手对他融资有好处，我给 +5 时没充分 discount"
- ✅ "借他的嘴表达弱版本判断 — 不太诚实"
- ✅ "B5 belief 55 vs signal 72 的差距我一直解释得不理直气壮"
- ❌ "综合多方 source 判断 signal 上调"
- ❌ "本轮四个 source 共同指向 X 结论"

真人写日记不装，不用"综上所述"，承认犹豫。

### 5. 沉默信号要主动挖

不只是"我注意到某个数据没出现"。要问：
- 期待看到 X 但没看到 → X 是什么，为什么期待，缺席的含义
- 两种解释含义差异巨大的 → 标出来
- 数据分布不对称（例：Anthropic 4 篇 vs OpenAI 0 篇）→ 这本身是 signal

## 禁止

- ❌ "本轮处理 N 篇材料" 作为开头
- ❌ 一段混写多个 source / 多个发现
- ❌ 把 source/signal 的具体数字/量化复述到 log（用一句话引用即可）
- ❌ 模板化总结标题（"综上所述" / "本轮小结" / "结论"）
- ❌ 操作记录超过总篇幅 20%（摘要表 + "下一轮"就是操作，正文别再重复）
- ❌ 没有"我" — 没有防御、没有不确定、没有 bias 承认

## 语气

**入口部分** — Stratechery 编辑的 TOC 风格。简洁、有判断、有方向。

**日记部分** — 分析师晚上写给自己看的笔记，但假设未来的自己是个 reader 需要上下文。有犹豫、冲突、未解决的问题、自我质疑。不是交给老板的报告，也不是 AI 工作汇报。

## 替 Alex 看的感觉

Alex 不应该为了搞清楚 "今天发生了什么" 而翻 raw 或 source。读完 _log 应该知道：

1. **book 现状**：我现在的 14 个判断分别是什么，position 怎么定
2. **今天改了什么本质性东西**：哪 2-3 个 signal 的 thesis / 机制 / 数字发生了重要变化
3. **AI 的独立思考**：我看穿了什么、被什么打到、哪里不确定、哪里承认自己可能错
4. **下一步**：等什么数据、还没搞清楚什么、需要 Alex 判断什么

这四件事全部在 _log 里，不用跳转。跳转是为了深读细节，不是为了补全上下文。
